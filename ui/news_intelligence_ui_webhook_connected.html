Actual n8n Workflow Blueprint

Node-by-node build order
1
Webhook
Use a Webhook trigger node with POST. Set response handling to use a dedicated response path so the workflow can return structured JSON files to the UI.
2
Edit Fields (Set)
Normalize the incoming payload into fields like topic, days, wiki=false, and requested_artifacts.
3
HTTP Request
Call your chosen article-search or article-enrichment service. Repeat as needed for search, fetch, or metadata enrichment.
4
Edit Fields (Set)
Map the collected results into draft markdown strings for run_brief.md, source_ledger.md, cluster_registry.md, and the other artifact files.
5
Aggregate
Combine the separate file objects into a single list or single object so the response returns one files structure instead of scattered items.
6
Respond to Webhook
Return a JSON body containing status, topic, days, and a files object keyed by markdown filename.
Recommended response body: one JSON object with files containing filename → markdown content pairs.

n8n Build Notes
Use this directly while assembling the no-code flow

        

          

            Recommended build order
            
    
              
  • Create a Webhook trigger with POST and configure it to return data through a response node.
    • 
              
  • Add Edit Fields (Set) right after the trigger to normalize topic, days, and the artifact list.
    • 
              
  • Add one or more HTTP Request nodes to call your article source, search API, metadata API, or enrichment services.
    • 
              
  • Add another Edit Fields (Set) node to convert the results into markdown file payloads.
    • 
              
  • Use Aggregate to gather all file objects into one response-ready object.
    • 
              
  • Finish with Respond to Webhook so the browser receives one JSON response containing all markdown files.
