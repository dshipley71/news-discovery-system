# Citation Traceability Specification

## Purpose
Define how citations are generated by the backend (not UI-derived) and linked to canonical evidence.

## Source of Truth
Citation records are produced in `stages.citation_traceability.citations[]` from `normalization.canonical_articles[]` after ingestion deduplication.

## Citation Record Contract
Each citation includes:
- `citation_id` (deterministic)
- `article_id` (canonical)
- `cluster_id`
- `claim_classification` (`supported|inferred|speculative`)
- `url`
- `source`
- `source_label`
- `title`
- `published_at`

## Classification Logic
Current implemented logic:
- Use upstream `claim_classification` when already present.
- Else:
  - `supported` when URL and publication timestamp are present.
  - `inferred` when title is present but support metadata is incomplete.
  - `speculative` otherwise.

This logic is deterministic and testable.

## Traceability Paths (Implemented)
- `cluster_id -> cluster.article_ids -> citation.article_id`
- `peak_day -> evidence.peak_to_clusters_articles[].clusters[].article_ids -> citations`
- `location_label -> evidence.location_to_clusters_articles[].article_ids -> citations`

## Attribution Preservation
Citation payload preserves article URL, source, title, and publication time where available.

## Deferred
- Claim statement-level citation binding (claim node graph) is deferred.
